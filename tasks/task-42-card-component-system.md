# Task 42 – Card Component System

Goal

Establish a reusable card component system for repeated content blocks.

Scope

Shared card surfaces used across achievements, founder message, business units, partnerships, and timeline blocks.

Files to Modify

index.html
styles.css

Changes

Add reusable card classes for base surface, spacing, and optional hover behavior.

Apply the reusable classes to existing card-based sections instead of relying on long grouped selectors alone.

Keep the visual treatment aligned with design-system.md tokens for background, border, radius, padding, and shadow.

Constraints

Do not introduce new card colors outside existing tokens.

Do not remove existing content or change section order.

Avoid duplicating card rules already present in styles.css.

Acceptance Criteria

Repeated content blocks share a visible base card class.

Interactive cards use a consistent hover treatment.

Card spacing, radius, and shadow remain consistent with design-system.md.
