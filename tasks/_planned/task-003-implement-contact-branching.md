# Task 003: Implement Contact Form Branching

## Title
Implement branching logic/UI on the contact page for different service inquiries.

## Problem
All inquiries currently go to a single generic contact point, making it difficult for both users and staff to route specialized health, care, or legal inquiries.

## Target Files
- `contact/index.html`

## Implementation
- Create a "Select Inquiry Type" UI or distinct contact blocks for:
    - Visiting Acupuncture (訪問施術)
    - Care Management (居宅介護支)
    - Fitness (べるフィット)
    - Recruitment (採用)
- Provide specific contact details (Phone/Mail) for each branch if applicable.

## Acceptance Criteria
- [ ] Users can clearly choose their inquiry type.
- [ ] Direct contact info for specific services is visible.

## Dependencies
- None.

## Priority
HIGH
