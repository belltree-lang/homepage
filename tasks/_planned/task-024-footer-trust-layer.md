# Task 024 – Footer Trust Layer

## Goal
フッターを信頼形成の補助レイヤーとして強化する

## Scope
変更可能ファイル
- index.html
- styles.css

## Changes
具体的変更
- 会社情報、所在地、問い合わせ、補助導線の優先順位を整理する
- フッター内の情報ブロックを視覚的に統一する
- ブランドの安心感を補強する文脈を追加する

## Constraints
禁止事項
- framework追加禁止
- design token必須
- semantic HTML維持
- フッターを情報過多にしない

## Acceptance Criteria
完了条件
- フッターに信頼情報が過不足なく配置されている
- 主要導線と会社情報が即座に見つかる
- フッター全体の視覚品質がページ本体と揃っている

## Dependencies
依存タスク
- Task 015 – Footer Redesign

## Parallel Safety
衝突しない範囲
- フッター専用変更であれば並行可能
