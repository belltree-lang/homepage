# Task 011 – Hero Visual Design

## Goal
ヒーローの視覚表現を強化し、企業サイトとしての第一印象を改善する

## Scope
変更可能ファイル
- index.html
- styles.css
- assets/images/*

## Changes
具体的変更
- ヒーローの背景、装飾、図形要素を使って印象を強化する
- テキスト可読性を保ちながら視覚的な奥行きを追加する
- 必要であれば既存アセットを追加または差し替える

## Constraints
禁止事項
- framework追加禁止
- design token必須
- semantic HTML維持
- 装飾が主張しすぎて本文可読性を下げない

## Acceptance Criteria
完了条件
- ヒーローに視覚的な主役感がある
- テキストが常に読みやすい
- 装飾要素がモバイルでも破綻しない

## Dependencies
依存タスク
- Task 007 – Hero Foundation
- Task 002 – Design Token Freeze

## Parallel Safety
衝突しない範囲
- ヒーロー専用スタイルと画像に限定すれば並行可能
