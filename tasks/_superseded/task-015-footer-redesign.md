# Task 015 – Footer Redesign

## Goal
フッターを信頼補強の役割を持つ情報レイヤーとして再設計する

## Scope
変更可能ファイル
- index.html
- styles.css

## Changes
具体的変更
- 会社情報、導線、連絡先、著作権表示の整理を行う
- フッター内の階層と余白設計を見直す
- ページ末尾として自然に締まる視覚表現へ調整する

## Constraints
禁止事項
- framework追加禁止
- design token必須
- semantic HTML維持
- 情報量を増やしすぎて冗長にしない

## Acceptance Criteria
完了条件
- フッター内の情報分類が明確である
- 連絡先と会社情報が容易に見つかる
- ページ末尾としての視覚的なまとまりがある

## Dependencies
依存タスク
- Task 010 – Button System

## Parallel Safety
衝突しない範囲
- フッターHTML/CSSに限定すれば他セクション作業と並行可能
