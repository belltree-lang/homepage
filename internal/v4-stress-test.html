診断結果（v4 を本番転用した場合の問題）

CRITICAL

HERO の 3列固定グリッド

  • 原因: grid-template-columns: 220px 1fr 240px
  • モバイル: 横スクロール発生、縦書きラベルが折れる、写真が縮みすぎる
  • シニア: 主見出し「人生の後半を…」が小さく可読性 NG
  • 対処: @media で 1column 化、装飾は非表示か再配置
CRITICAL

SERVICES カードの装飾被り

  • 原因: 装飾を position:absolute; bottom:-18px; right:-12px で写真に重ねる
  • モバイル: カード幅縮小で写真が小さくなり、装飾が写真の半分を覆う
  • シニア: サービス名「訪問鍼灸マッサージ」が一部見えなくなる可能性
  • 対処: @media で装飾サイズを縮小、もしくは display:none
CRITICAL

PULL QUOTE の巨大引用符

  • 原因: font-size: 280px 固定の ❝ ❞
  • モバイル: 引用符が画面より大きくなり、引用テキストが完全に隠れる
  • 対処: clamp(80px, 24vw, 280px) でレスポンシブ化
CRITICAL

MESSAGE / TIMELINE の固定 column

  • 原因: MESSAGE は 240px 1fr 80px、TIMELINE は repeat(5, 1fr)
  • モバイル: 代表 portrait と本文が重なる、TIMELINE は 5列が極小に
  • 対処: @media で縦並びへ
WARNING

FOOTER 4列固定

  • 原因: 1.4fr 1fr 1fr 1.6fr
  • モバイル: 各列が極小、住所・電話番号が読めない
  • 対処: @media で 2列 or 1列に
WARNING

パフォーマンス影響

  • 装飾画像: HERO + SERVICES + CASES + MESSAGE + COMMUNITY で約 12〜15 枚追加
  • 容量: 合計 1〜2MB の追加リソース、初回ロードに 1〜2 秒影響
  • シニア環境: 低速回線で体感的に遅くなる
  • 対処: HERO 以外は loading=lazy、サムネ縮小（256×256 等）
WARNING

既存スタイルとの衝突

  • 原因: v4 は HTML 内 <style>、本番は外部 styles.css
  • クラス名衝突: .service-rich 等の汎用名は既存と被るリスク
  • 対処: .v4- プレフィックスで分離、もしくは規格 class を styles.css に追加
NOTE

シニア可読性

  • 本文サイズ: 15-16px 中心で OK だが、装飾が散らばると視線誘導が崩れる
  • 明朝の小サイズ: PULL QUOTE のキャプション 13px 明朝は読みにくい
  • 対処: 装飾は写真の角に限定し、本文周辺に置かない

判定 — 本番転用は段階を踏むべき

v4 はあくまで PC 幅前提の世界観確認用モック。本番 index.html はレスポンシブが前提で、シニア層がモバイルから多くアクセスするため、以下の Step 1（装飾 CSS のレスポンシブコンポーネント化）を経ないと本番反映できません。具体的には @media (max-width: 768px) で装飾を縮小・非表示にする規格を styles.css に先に組み、その上で index.html に流し込むのが安全。
