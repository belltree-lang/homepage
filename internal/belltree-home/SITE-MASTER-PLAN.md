対象: `internal/belltree-home/`（べるつりー法人サイト）
状態: Phase 1（骨格完成）に着手可 — Phase 0 完了（2026-06-11 代表決定）

## Phase 0 決定事項（2026-06-11 代表決定）

- **掲載先: 案A採用** — `belltree1102.com` で法人コーポレートサイトとして独立公開
- bellmfit.com はべるフィット専用サイトに純化（本件完了後の後続プロジェクト）
- SEO論点の整理: ドメイン名一致の直接効果はなし。指名検索はGBP整備で受ける。新ドメインの助走期間（数ヶ月）はGBP・チラシQR・紹介経由で補う
- **技術注意**: belltree1102.com はメール（Google Workspace）で使用中。Web公開のDNS設定時にMXレコードを壊さないこと（Phase 3）

## 1. このホームページは「誰のための、何か」

優先順で3つの顔を持つ。迷ったら上位の顔を優先する。

| 順 | 顔 | 具体的な場面 |
|---|---|---|
| 1 | **営業の受け皿** | チラシ・パンフ・ケアマネの紹介で名前を知った人が、検索して確認し、安心する場所。営業3本柱（リファラル／病院連携室／断られた後の継続接点）はすべて最後にここを通る |
| 2 | **問い合わせの獲得** | 「親が転んだ」「退院後のリハビリがない」で検索した家族・本人を、無料体験の申込みへつなぐ |
| 3 | **会社の顔** | 採用応募者・銀行・取引先・行政が見る信頼の土台 |

## 2. ゴール（数値・たたき台）

幹部会決定「訪問鍼灸 年間+100名（月10名流入）」のうち、HPが担う分を定義する。

- **公開後3ヶ月**: 計測を整え、HP経由の問い合わせ（電話タップ＋フォーム）月5件
- **公開後6ヶ月**: HP経由の無料体験 月3件 → 利用開始 月2名（年間換算24名 ≒ 100名目標の1/4をHPが担う）
- 成果（コンバージョン）の定義: ①電話タップ ②無料体験フォーム送信 ③専門職ページからの連携相談

※数値は代表が調整。計測できる状態を先に作るのがPhase 3。

## 5. 全体の流れ（5フェーズ）

```
Phase 0  戦略確定 ────────── 掲載先・ドメイン（代表判断）／ゴール数値の確定
Phase 1  骨格完成 ────────── サービス詳細3枚＋専門職ページ＋料金保険ページ → リンク切れ解消
Phase 2  転換装置 ────────── 無料体験フォーム＋電話導線＋プライバシーポリシー
Phase 3  公開準備 ────────── fact-check全頁 → SEO/OGP → GA4 → 本番デプロイ
Phase 4  運用 ──────────── 事例追加・GBP連携・チラシQR連動・月次の数字確認
```

### Phase 1: 骨格完成（Claude実装・素材は既存流用）

| 成果物 | 中身 | 流入元 |
|---|---|---|
| 訪問鍼灸マッサージ詳細 | 施術内容（鍼・お灸・マッサージ）／保険のしくみ／施術の流れ／エリア | 解決策7ページの「詳細を見る」 |
| 居宅介護支援 詳細 | ケアマネの役割／相談無料／介護の始め方ページと連動（公正中立の表現遵守） | 介護の始め方ほか |
| べるフィット詳細 | 「保険が使えるジム」表記ルール遵守 | 膝・フレイル・転倒 |
| 専門職向け連携案内 | ケアマネ・病院連携室向け。媚びない事実列挙。紹介の流れ・報告のしかた | 全ページのpro-cta |
| 料金と保険 | 帯③で触れた費用不安への本回答。症状別の保険適用・自己負担のめやす・無料体験 | 全ページ |

### Phase 2: 転換装置

- 無料体験・相談フォーム（最小項目: 名前・電話・地域・困りごと。入力1分）
- スマホ追従の電話CTA確認・タップ計測
- プライバシーポリシー（フォーム設置に必須）

### Phase 3: 公開準備

- fact-check（3兄弟ゲート）全ページ → 修正
- title/description/OGP整備、構造化データ（地域サービス）
- GA4＋電話タップ・フォーム送信イベント
- 本番ドメイン設定 → デプロイ → リダイレクト・サーチコンソール登録

### Phase 4: 運用（公開後）

- 改善事例を月1本追加（現場の「瞬間」を帯と同じ書き方で）
- GBP（Googleビジネスプロフィール）と相互連携
- チラシ・パンフの次回刷にHPのQRを載せる
- 月次：問い合わせ数・流入元を確認 → 弱いページから直す

## 6. 先に決めること（Phase 0・代表判断）

**掲載先・ドメイン**（これが決まらないとPhase 3の設計が組めない）

| 案 | 内容 | 利点 | 難点 |
|---|---|---|---|
| A | 法人ドメインで独立公開（例: belltree1102.com） | メールと同じドメインで信頼一致。bellmfit.com（べるフィット）と役割分担が明快 | DNS・ホスティング設定が新規 |
| B | bellmfit.com のトップを置き換え | 既存ドメインの実績を引き継ぐ。設定が軽い | べるフィット事業サイトと法人サイトが混ざる |
| C | bellmfit.com 配下に法人サイトを同居 | 作業最小 | URLが長く営業ツールとして弱い。指名検索で迷子になりやすい |

推奨は **案A**。営業の受け皿（顔1）として「べるつりーで検索→法人ドメイン」が一番まっすぐで、既存のbellmfit.comはべるフィットの集客サイトとして残せる。
